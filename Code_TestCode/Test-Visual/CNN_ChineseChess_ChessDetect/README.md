# 中国象棋棋子识别

基于 OpenCV 霍夫圆检测定位棋子 + PyTorch CNN 分类棋子文字类别，适用于 1080×1080 俯拍摄像头画面。

## 项目结构

```
aa/
├── cam_hough_circle.py   # 原始霍夫圆检测 demo（仅检测，不分类）
├── data_collector.py     # PyQt5 数据采集工具（手动标注棋子类别）
├── train_cnn.py          # CNN 训练脚本
├── dataset/              # 采集的图片数据（自动生成）
│   ├── 将帅/
│   ├── 士仕/
│   ├── 車/
│   ├── 馬/
│   ├── 炮/
│   ├── 象相/
│   └── 兵卒/
├── chess_cnn.pt          # 训练输出的模型权重（训练后生成）
├── train_history.json    # 训练记录（训练后生成）
└── train_curve.png       # 训练曲线图（训练后生成）
```

## 环境准备

```bash
pip install opencv-python numpy PyQt5 torch torchvision matplotlib
```

> 如果有 NVIDIA 显卡并想用 GPU 加速训练，请安装对应 CUDA 版本的 PyTorch，参考 [PyTorch 官网](https://pytorch.org/)。

## 第一步：采集数据（data_collector.py）

### 启动

```bash
python data_collector.py
```

程序会打开摄像头窗口，实时显示 1080×1080 画面并用紫色圆圈标出检测到的棋子。

### 操作方式

| 操作 | 说明 |
|---|---|
| **鼠标左键点击棋子** | 弹出类别选择窗口，左侧预览该棋子截图 |
| **数字键 1 ~ 7** | 在弹窗中快速选择对应类别 |
| **Enter** | 确认保存 |
| **Esc** | 取消标注 |
| **Space** | 暂停 / 继续摄像头画面 |
| **Esc（主窗口）** | 退出程序 |

### 7 个类别

| 按键 | 类别 | 说明 |
|---|---|---|
| 1 | 将帅 | 红方的"帅"、黑方的"将"归为一类 |
| 2 | 士仕 | 红方的"仕"、黑方的"士"归为一类 |
| 3 | 車 | 双方的"車"（車/车） |
| 4 | 馬 | 双方的"馬"（馬/马） |
| 5 | 炮 | 双方的"炮" / "砲" |
| 6 | 象相 | 红方的"相"、黑方的"象"归为一类 |
| 7 | 兵卒 | 红方的"兵"、黑方的"卒"归为一类 |

> 红黑方的区分不通过 CNN 分类，而是后续通过棋子表面的**颜色阈值**（红色 / 黑色）来判断。CNN 只需识别文字类别。

### 采集建议

1. **摆放好摄像头和棋盘**，确保光照稳定、棋盘正对摄像头
2. 每个类别至少采集 **50 ~ 100 张**，越多越好
3. 尽量覆盖不同位置、角度的棋子（移动棋盘位置再采集几轮）
4. 底部状态栏实时显示每个类别已采集的数量
5. 采集的图片自动保存到 `dataset/` 目录下，每张 128×128 像素

## 第二步：训练模型（train_cnn.py）

### 需要手动划分训练集 / 测试集吗？

**不需要。** `train_cnn.py` 会自动将数据按 **80% 训练 / 20% 验证** 随机划分（可通过脚本中 `TRAIN_RATIO` 参数调整）。你只需要把所有采集的图片放到 `dataset/` 的各类别文件夹里即可。

### 启动训练

```bash
python train_cnn.py
```

### 训练过程

程序会输出每个 epoch 的训练和验证指标：

```
[  1/60] Train Loss: 2.1543 Acc: 0.1250 | Val Loss: 1.9876 Acc: 0.1667 | LR: 0.001000 | 2.3s *best*
[  2/60] Train Loss: 1.8234 Acc: 0.2917 | Val Loss: 1.6543 Acc: 0.3333 | LR: 0.001000 | 2.1s *best*
...
```

| 字段 | 含义 |
|---|---|
| Train Loss / Acc | 训练集的损失和准确率 |
| Val Loss / Acc | 验证集的损失和准确率 |
| LR | 当前学习率（自动衰减） |
| `*best*` | 该轮验证集 loss 最低，模型被保存 |
| 早停提示 | 验证集连续 10 轮不下降时自动停止训练 |

### 训练输出

训练结束后自动生成三个文件：

| 文件 | 说明 |
|---|---|
| `chess_cnn.pt` | 模型权重文件，用于后续推理 |
| `train_history.json` | 每个 epoch 的 loss/acc 数值记录 |
| `train_curve.png` | Loss 和 Accuracy 的变化曲线图 |

### 训练技巧

- **数据量不够时**：先每类采集 30 张跑通流程，再逐步增加到 100+
- **过拟合迹象**：训练准确率很高但验证准确率低 → 增加数据量或减少 epoch
- **欠拟合迹象**：训练和验证准确率都很低 → 检查数据标注是否正确、增加 epoch
- **调整超参数**：在 `train_cnn.py` 顶部的配置区域修改 `EPOCHS`、`LR`、`BATCH_SIZE` 等
- **使用 GPU**：如果安装了 CUDA 版 PyTorch，训练会自动使用显卡，速度提升数倍

## 关键参数说明

| 参数 | 位置 | 默认值 | 说明 |
|---|---|---|---|
| `CAM_INDEX` | data_collector.py | 0 | 摄像头编号（0=默认，1=第二个） |
| `HOUGH_PARAM2` | data_collector.py | 25 | 霍夫圆检测阈值，越小检测越灵敏 |
| `SAVE_SIZE` | data_collector.py | 128 | 保存图片尺寸 |
| `IMG_SIZE` | train_cnn.py | 128 | 训练输入尺寸（需与 SAVE_SIZE 一致） |
| `BATCH_SIZE` | train_cnn.py | 32 | 批大小（显存不够可减小） |
| `EPOCHS` | train_cnn.py | 60 | 最大训练轮数 |
| `LR` | train_cnn.py | 0.001 | 初始学习率 |
| `PATIENCE` | train_cnn.py | 10 | 早停耐心值 |
| `TRAIN_RATIO` | train_cnn.py | 0.8 | 训练集占比 |
